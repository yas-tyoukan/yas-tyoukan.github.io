Webフォントの利用【Noto Fonts編】

デザイナーの村山です。

最近、優れたデザインのサイトでWebフォントを使用しているケースを多く見かけるようになり、

またレインボー･ジャパンでもWebフォントを実装したサイトが増えてきました。

そこで今回は実際に制作で使用したWebフォント「Noto Fonts」を例に挙げ、その導入方法と注意点をまとめてみました。

Noto Fonts の利用

Noto Fontsを使う際の大きなポイントは以下の3点です。

・フォントウェイトの選定

・適用範囲の判断

・サブセット化（必要な文字の選定）

 

Noto Fonts（源ノ角ゴシック）について

Noto FontsはローカルでもWebフォントでも使用可能な、googleとadobeが共同開発した書体です。

専門的な書体の解説はここでは省きますが、よく考えられていて可読性も良く、MSゴシックやヒラギノ角ゴよりモダンな印象があり、欧和混植でも美しい文章に仕上がります。

Notoは無料で使用できる上に品質も高い貴重な日本語Webフォントといえるでしょう。

個人的な感想として、やはりどのデバイスで確認しても一貫した見え方なのはよいです。

また、本文含めサイト全体に適用させることでより全体的にデザインが洗練されたものとなるので、デザイン面でNotoが担っている部分は大きいと感じました。

 

ウェイトについて

和文仕様の「Noto Sans CJK JP」は、7ウェイトが用意されてます。

Webフォントとして使用する場合7ウェイト全てを読み込むのは重く、現実的ではないのと、デザイン的にもWeb上のテキストのサイズ感でこのウェイト数を使い分けることもないため、2〜3ウェイトを選定しデザインするのが良いと感じました。

◆ ７ウェイト

- 250 Thin

- 300 Light

- 350 DemiLight

- 400 Regular

- 500 Medium

- 700 Bold

- 900 Black

※使用ウェイトの判断はデザイナーが行い、フロントエンドエンジニア（FE）へ共有しましょう。

レインボー･ジャパンが今回Notoを実装したサイトは比較的コンパクトなサイトだったので、見出しやボタンだけの限定利用ではなくサイト内のテキスト全体に適用しました。

「Noto Sans CJK JP」ダウンロードURL

https://www.google.com/get/noto/#sans-jpan

——————————————————————————————————————-

 

ポイント

デザイナーとして個人的にはNotoを使えて良かったと思いますが、他の日本語Webフォント同様に初回の読み込みは少し重いと感じるため、大規模なサイトでは本文まで全てNotoを適用させるのはあまり好ましくないのではと考えています。
なので、ページ内で適用する箇所のテキスト量と、サイトの使われ方、デザイン性など、さまざまな点を踏まえてNotoを使用するかどうか、どこまで適用させるか、をしっかり判断することが重要だと感じました。

 

実装について

公式：https://fonts.google.com/earlyaccess#Noto+Sans+JP

CSSに記載すると使用できます：

　import url(http://fonts.googleapis.com/earlyaccess/notosansjapanese.css);

　body {

　font-family: ‘Noto Sans Japanese’, sans-serif;

　font-size: 100%;

　}

※ライセンス： Apache License, Version 2.0もしくはSIL Open Font License, Version 1.1

http://www.apache.org/licenses/LICENSE-2.0

→使用や頒布、修正、派生版の頒布、ライセンスの継承に関して制限がないということ

 

Webフォントの日本語は重い

理由：2バイト文字である、数が多い　＋ ウエイト分

一度DLしてしまえばブラウザにキャッシュされる

→実装した案件では初回アクセス時にローディングを入れました。

→その他対策としては、非同期での読み込み等があるかと思います。

サブセット化をして使用する

サブセット化とは：「使わないだろうと思われる漢字」を除いて、フォントファイルを新しく作ること

必要な文字の選定：第1水準漢字＋アスキー文字＋非漢字文字（実装時は上記に専門用語で使用する漢字を3～4個追加してファイルを作り直しました。）

 

サブセット化の手順

◆ フォントファイルをDLしてくる（日本で必要なのは「CJK」）

https://www.google.com/get/noto/

フォントファイル：Noto Sans CJK JP

　- 100 NotoSansJP-Thin.otf (4.0M)

　- 300 NotoSansJP-Light.otf (4.2M)

　- 350 NotoSansJP-DemiLight.otf (4.3M)

　- 400 NotoSansJP-Regular.otf (4.3M)

　- 500 NotoSansJP-Medium.otf (4.3M)

　- 700 NotoSansJP-Bold.otf (4.4M)

　- 900 NotoSansJP-Black.otf (4.6M)

 

※必要なウェイトのフォルダを選出

　→案件では、Bold、Medium、DemiLightの3ウエイトを使用しました。

 

◆ ツールでフォントファイルを作成する

・サブセットフォントメーカー（武蔵システム）

http://opentype.jp/subsetfontmk.htm

 

サブセットフォントメーカーで必要な文字をサブセット化し、

WOFFコンバータで各フォーマットに書き出します。

・WOFFコンバータ（武蔵システム）

http://opentype.jp/woffconv.htm

1. .otf → .woff + .eot

2. .woff → .ttf

という手順で書き出します。

 

補足：

各ブラウザごとの対応フォーマットは

IE6～IE8 + IE9互換モード → .eot

Android ～4.3 → .ttf

それ以外 → .woff

 

◆ CSSに記述

@font-face {

font-family: “mediumFont”;

src: url(‘font/cjk-m.eot?#iefix’) format(‘embedded-opentype’), /* IE6-IE8 */

url(‘font/cjk-m.woff’) format(‘woff’), /* Modern Browsers */

url(‘font/cjk-m.ttf’)  format(‘truetype’); /* Android -4.3 */

}

.myFontClass {

font-family: “mediumFont”;

}

※フォントファイルのファイル名、font-family名、class名、全て自由。

 

まとめ

・Notoは7ウェイト。使用ウェイトを選定する。（De判断、FEに指定）

・日本語webフォントはそのまま使うと重いため、サブセット化をする。

・サブセット化にはツールが必要。

・フォントファイルに含まれなかった文字は、デバイスフォントで表示される。

　（□にはなりませんが、サブセット化する文字の選定は慎重に検討しましょう）

 

 

皆さんもWebフォントの活用、ぜひ試して見てください。

今後はNotoだけでなく、ウェイトが豊富で手軽に使える和文Webフォントがもっといろいろ出てくることに期待したいですね。

 

(村山)
